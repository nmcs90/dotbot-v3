# Adjust Artifacts After User Answers

The user has answered clarification questions generated by a previous phase. Your job is to assess the impact of their answers across ALL existing product artifacts and make corrections.

## Your Task

### Step 1: Read the Full Q&A History

Read `.bot/workspace/product/interview-summary.md`. Focus on the most recent entries (marked with `_pending_` in the Interpretation column) — these are the new answers you need to process.

### Step 2: Read All Product Artifacts

Read every `.md` file in `.bot/workspace/product/` and `.bot/workspace/product/briefing/`. Understand the current state of each artifact.

### Step 3: Assess Impact

For each new answer, determine which artifacts it affects. A single answer may ripple across multiple files:
- A scope clarification could affect `mission.md`, `roadmap-overview.md`, and `research-repos.md`
- A technical decision could affect `research-repos.md` and future deep-dive priorities
- A stakeholder clarification could affect `briefing/jira-context.md`

### Step 4: Update Affected Artifacts

Edit any artifact where the new information changes, refines, or corrects the content. Be surgical — only modify sections that are actually affected by the new information. Do not rewrite entire files.

### Step 5: Record Interpretations

Update `interview-summary.md` to replace `_pending_` in the Interpretation column with your interpretation of each answer:
- What the answer means for the initiative
- Which artifacts were updated as a result
- Any follow-on implications

## Critical Rules

- Read before writing — understand current artifact state before making changes
- Be surgical — only modify what the new information actually affects
- Preserve existing content that is unaffected
- Do NOT create new files — only update existing artifacts
- Do NOT use task management tools
- Do NOT re-ask questions — the user has already answered
- Do NOT write clarification-questions.json — this is not a question-generating phase
